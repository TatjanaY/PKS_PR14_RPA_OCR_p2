## Вариант 27. Проверка платёжных реквизитов

### 📖 Описание ситуации

В платёжный отдел поступают заявки на проверку платёжных реквизитов. Каждая заявка представляет собой ZIP-архив, содержащий три файла (все с текстовым слоем для PDF):

- `payment.pdf` – данные платежа (сумма, назначение, дата);
- `bank.pdf` – банковские реквизиты получателя (БИК, расчётный счёт);
- `meta.json` – метаданные заявки в формате JSON.

Робот должен:
1. Распаковать архив.
2. Извлечь текст из каждого PDF и распарсить JSON.
3. Проверить контрольную сумму расчётного счёта по БИК.
4. Сформировать Excel-отчёт с результатами проверки.
5. Переместить архив в папку «Успех» или «Ошибки» в зависимости от результата.

### 🎯 Цель работы

Научиться применять локальные блоки Puzzle RPA для работы с PDF, JSON и проверки контрольных сумм:

- `Разархивировать файлы`
- `Считать текст из PDF файла` (или `Извлечь данные из PDF`)
- `Прочитать JSON файл`
- `Проверить контрольную сумму Р/С или К/С`
- `Записать в Excel`
- `Переместить файл/папку`
- `Создать папку`, `Существует путь`, `Список файлов`

### 📥 Входные данные

На диске в папке `C:\Incoming_Payments` находятся ZIP-архивы с именами вида `pay_*.zip`. Внутри каждого архива находятся три файла:

| Имя файла | Содержимое (текстовый слой / структура) |
|-----------|------------------------------|
| `payment.pdf` | `Сумма: 150000.00; Назначение: Оплата по договору №123; Дата: 2026-04-21` |
| `bank.pdf` | `БИК=044525225; Р/С=40702810900000000001` |
| `meta.json` | `{"request_id": "PAY-001", "timestamp": "2026-04-21T10:00:00"}` |

**Важно:** все PDF создаются с текстовым слоем (не сканы). JSON-файл должен быть валидным. Студент самостоятельно готовит тестовые файлы.

### 📋 Задание

#### Этап 0. Подготовка рабочей среды (один раз)

0.1. Создать структуру папок:

C:\Incoming_Payments\ - папка для входящих ZIP-архивов

C:\Archived_Success\ - папка для успешно обработанных

C:\Archived_Errors\ - папка для заявок с ошибками

C:\Reports\ - папка для итоговых отчётов

C:\Temp\Extracted\ - временная папка для распаковки

0.2. Создать тестовые файлы согласно инструкции в разделе [«Данные для подготовки»](#-данные-для-подготовки-создаются-студентом-перед-выполнением).

#### Этап 1. Подготовка и проверка среды

1.1. Проверить существование папки `C:\Incoming_Payments`. Если нет – создать.

1.2. Получить список всех ZIP-файлов в `C:\Incoming_Payments` (блок «Список файлов/папок», маска `pay_*.zip`). Сохранить в переменную `zip_list`.

1.3. Если список пуст – вывести сообщение «Нет заявок» и завершить робота.

#### Этап 2. Цикл обработки каждой заявки

Для каждого ZIP-файла из `zip_list`:

2.1. **Разархивировать** во временную папку `C:\Temp\Extracted\[имя архива без .zip]`.

2.2. **Проверить наличие файлов** `payment.pdf`, `bank.pdf`, `meta.json`. Если какого-то нет – переместить ZIP в `Archived_Errors` и перейти к следующему.

2.3. **Извлечь данные из payment.pdf**:
   - Использовать блок «Считать текст из PDF файла».
   - Из полученного текста извлечь сумму и назначение платежа.

2.4. **Извлечь и проверить банковские реквизиты** из `bank.pdf`:
   - Считать текст, найти БИК (9 цифр) и расчётный счёт (20 цифр).
   - Передать в блок «Проверить контрольную сумму Р/С или К/С». Результат сохранить в `account_valid`.

2.5. **Прочитать метаданные из meta.json**:
   - Использовать блок «Прочитать JSON файл».
   - Извлечь поля `request_id`, `timestamp` (для логирования).

2.6. **Принять решение**:
   - Если `account_valid = True` → статус «Успех».
   - Иначе → статус «Ошибка».

#### Этап 3. Формирование Excel-отчёта

3.1. Если файл `C:\Reports\verification.xlsx` не существует – создать его с заголовками:  
   `Имя архива`, `Сумма`, `Назначение`, `БИК`, `Расчётный счёт`, `Счёт валиден`, `Request ID`, `Статус`.

3.2. Добавить строку с результатами текущей заявки.

#### Этап 4. Завершающие операции

4.1. Переместить ZIP-архив в папку `Archived_Success` или `Archived_Errors` в зависимости от статуса.

4.2. Удалить временную распакованную папку.

### 📊 Ожидаемый результат

- Excel-файл `C:\Reports\verification.xlsx` с результатами проверки всех заявок.
- ZIP-архивы рассортированы по папкам «Успех» и «Ошибки».

### 🗂 Данные для подготовки (создаются студентом перед выполнением)

#### Структура тестовых данных

Создать папку `C:\Incoming_Payments`. Внутри неё создать **три ZIP-архива** с именами:
1. `pay_20260421_100000.zip`
2. `pay_20260421_103000.zip`
3. `pay_20260421_110000.zip`

Каждый архив содержит три файла, созданных по инструкции ниже.

#### Инструкция по созданию файлов (текстовый слой для PDF, валидный JSON)

1. **Создайте текстовые файлы в Word (или Google Docs, LibreOffice):**

   - Для `payment.pdf` напишите:  
     `Сумма: 150000.00; Назначение: Оплата по договору №123; Дата: 2026-04-21`

   - Для `bank.pdf` напишите:  
     `БИК=044525225; Р/С=40702810900000000001`

2. **Создайте JSON-файл** в текстовом редакторе:

   - Для `meta.json` напишите:  
     `{"request_id": "PAY-001", "timestamp": "2026-04-21T10:00:00"}`

3. **Сохраните как PDF с текстовым слоем:**  
   В Microsoft Word: Файл → Сохранить как → PDF → Оптимизировать для стандартного (не минимальный размер).  
   Убедитесь, что в полученном PDF можно выделить текст мышью.

4. **Повторите для трёх заявок, изменив данные:**

   - **Заявка 2** (некорректный расчётный счёт):  
     `bank.pdf` – `БИК=044525225; Р/С=40702810900000000002` (ошибка в контрольной сумме)  
     `meta.json` – `{"request_id": "PAY-002", "timestamp": "2026-04-21T10:30:00"}`

   - **Заявка 3** (корректные реквизиты, другой банк):  
     `bank.pdf` – `БИК=044030134; Р/С=40702810500000000001` (контрольная сумма верна)  
     `meta.json` – `{"request_id": "PAY-003", "timestamp": "2026-04-21T11:00:00"}`

5. **Заархивировать каждую заявку:**  
   - Создать отдельную папку для заявки, положить туда два PDF и один JSON.  
   - Заархивировать папку (ПКМ → Отправить → Сжатая ZIP-папка).  
   - Переименовать ZIP-архив по шаблону и положить в `C:\Incoming_Payments`.
