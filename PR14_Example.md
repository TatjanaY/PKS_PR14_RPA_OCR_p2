# Вариант 0

# Автоматизация веб-приложения с использованием Machine Vision (Puzzle RPA)

---

# 📖 Ситуация

Компания **EduStreamLab** использует веб-портал для регистрации и обработки заявок на участие в образовательных программах.

Система была разработана на предыдущих этапах обучения и содержит:

* страницу авторизации;
* форму подачи заявки;
* таблицу заявок;
* элементы управления статусами заявок.

В некоторых корпоративных средах веб-приложение может запускаться:

* через **виртуальные рабочие станции**;
* через **удалённые браузеры**;
* внутри **Citrix / RDP-сессий**.

В таких условиях традиционная веб-автоматизация (XPath, CSS-селекторы) может быть недоступна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение веб-страницы;
* распознаёт текстовые элементы интерфейса;
* взаимодействует с элементами по координатам.

В рамках данной практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, который выполняет автоматизацию веб-приложения **только через методы компьютерного зрения**.

---

# 🖥 Используемое веб-приложение

Веб-портал состоит из следующих страниц:

### 1. Страница авторизации

### 2. Страница подачи заявки

### 3. Таблица заявок участников

### 4. Панель управления заявками

---

# Страница авторизации

Назначение страницы — **проверка пользователя и предоставление доступа к системе**.

---

## Поле `login`

Назначение элемента:

используется для ввода имени пользователя.

Робот должен вводить значение:

```
admin
```

---

## Поле `password`

Назначение элемента:

используется для аутентификации пользователя.

Робот вводит пароль:

```
admin123
```

---

## Кнопка `Sign In`

Назначение элемента:

выполняет проверку введённых данных и выполняет переход на страницу подачи заявки.

После успешного входа на странице появляется форма заявки.

---

# Страница подачи заявки

Страница используется для **создания новой заявки участника мероприятия**.

Пользователь заполняет форму и отправляет её в систему.

---

## Поле `Participant Name`

Назначение:

имя участника мероприятия.

---

## Поле `Email`

Назначение:

контактный email участника.

---

## Поле `Event Name`

Назначение:

название мероприятия.

---

## Список `Participation Format`

Назначение:

выбор формата участия.

Возможные значения:

```
Online
Offline
```

---

## Чекбокс `Accept Terms`

Назначение:

подтверждение согласия пользователя с условиями участия.

Без установки чекбокса отправка формы невозможна.

---

## Кнопка `Submit Application`

Назначение:

создаёт новую запись заявки.

После отправки:

* появляется сообщение об успешной отправке;
* запись добавляется в таблицу заявок.

---

# Таблица заявок

Назначение таблицы — **отображение всех поданных заявок**.

Структура таблицы:

| ID | Name | Email | Event | Format | Status |

---

### Статус новой заявки

После создания заявка получает статус:

```
Pending
```

---

# Кнопки управления заявками

---

## `Mark as Approved`

Назначение:

изменяет статус выбранной заявки на

```
Approved
```

---

## `Mark as Rejected`

Назначение:

изменяет статус заявки на

```
Rejected
```

---

## `Delete Application`

Назначение:

удаляет выбранную заявку из таблицы.

# УРОВЕНЬ 1

# Базовая визуальная автоматизация веб-интерфейса

---

# Шаг 1. Запуск браузера и открытие страницы

### Зачем выполняется шаг

Необходимо открыть веб-приложение и убедиться, что страница полностью загрузилась.

---

### Что необходимо сделать

1. Использовать блок:

**Browser → Open Browser**

2. Перейти по адресу веб-приложения.

3. Выполнить поиск текста:

**Machine Vision → Find Text On Screen**

Искомый текст:

```
Sign In
```

Нахождение текста означает, что страница авторизации загружена.

---

# Шаг 2. Ввод логина

### Зачем выполняется шаг

Необходимо выполнить авторизацию пользователя.

---

### Что необходимо сделать

1. Использовать блок:

**Machine Vision → Find Text On Screen**

Текст:

```
login
```

2. Выполнить клик в поле ввода рядом с найденной меткой:

**Mouse → Click**

3. Ввести текст через:

**Keyboard → Type Text**

---

# Шаг 3. Ввод пароля

### Зачем выполняется шаг

Пароль необходим для аутентификации пользователя.

---

### Что необходимо сделать

1. Найти текст:

**Machine Vision → Find Text On Screen**

```
password
```

2. Кликнуть по полю ввода.

3. Ввести пароль:

```
admin123
```

через блок:

**Keyboard → Type Text**

---

# Шаг 4. Нажатие кнопки Sign In

### Зачем выполняется шаг

После ввода данных необходимо выполнить вход в систему.

---

### Что необходимо сделать

1. Использовать блок:

**Machine Vision → Find Text On Screen**

Текст:

```
Sign In
```

2. Выполнить:

**Mouse → Click**

---

# Шаг 5. Ожидание загрузки формы заявки

### Зачем выполняется шаг

После авторизации открывается страница формы.

Необходимо убедиться, что она загружена.

---

### Что необходимо сделать

Использовать:

**Machine Vision → Find Text On Screen**

Текст:

```
Participant Name
```

---

# Шаг 6. Заполнение формы заявки

### Зачем выполняется шаг

Робот должен автоматически создать заявки участников.

---

### Что необходимо сделать

Для каждой записи входного файла:

---

## 6.1 Заполнение Participant Name

Использовать:

**Machine Vision → Find Text On Screen**

```
Participant Name
```

Затем:

* клик по полю
* ввод текста

---

## 6.2 Заполнение Email

Использовать:

**Machine Vision → Find Text On Screen**

```
Email
```

Далее выполнить ввод значения.

---

## 6.3 Заполнение Event Name

Использовать:

**Machine Vision → Find Text On Screen**

```
Event Name
```

После нахождения выполнить ввод текста.

---

## 6.4 Выбор Participation Format

1. Найти элемент:

```
Participation Format
```

через

**Machine Vision → Find Text On Screen**

2. Кликнуть по списку.

3. Найти текст:

```
Online
или
Offline
```

4. Выполнить клик.

---

## 6.5 Установка чекбокса

Использовать:

**Machine Vision → Find Text On Screen**

```
Accept Terms
```

Кликнуть по чекбоксу.

---

# Шаг 7. Отправка формы

---

### Что необходимо сделать

1. Найти кнопку:

```
Submit Application
```

через

**Machine Vision → Find Text On Screen**

2. Выполнить:

**Mouse → Click**

# Шаг 8. Захват изображения таблицы

### Зачем выполняется шаг

Для анализа заявок необходимо получить изображение таблицы.

---

### Что необходимо сделать

Использовать блок:

**Screen → Capture Region**

Выделить область таблицы.

---

# Шаг 9. Распознавание текста таблицы

### Зачем выполняется шаг

Необходимо определить заявки по их параметрам.

---

### Что необходимо сделать

Использовать блок:

**Machine Vision → OCR Recognize Text**

Источник:

изображение таблицы.

---

# Шаг 10. Поиск заявок Online

### Зачем выполняется шаг

Заявки с форматом **Online** должны быть одобрены.

---

### Что необходимо сделать

Найти текст:

```
Online
```

через:

**Machine Vision → Find Text On Screen**

---

# Шаг 11. Одобрение заявки

### Что необходимо сделать

1. Кликнуть по строке таблицы.
2. Найти кнопку:

```
Mark as Approved
```

через

**Machine Vision → Find Text On Screen**

3. Выполнить клик.

---

# Шаг 12. Отклонение заявок Offline

1. Найти текст:

```
Offline
```

2. Выбрать строку.

3. Найти кнопку:

```
Mark as Rejected
```

4. Выполнить клик.

---

# Шаг 13. Удаление записи

### Зачем выполняется шаг

Необходимо проверить работу кнопки удаления.

---

### Что необходимо сделать

1. Выбрать любую строку таблицы.
2. Найти кнопку:

```
Delete Application
```

через

**Machine Vision → Find Text On Screen**

3. Нажать кнопку.

---

# 📊 Ожидаемый результат

После выполнения процесса:

* все заявки добавлены;
* заявки с `Format = Online` имеют статус:

```
Approved
```

* заявки с `Format = Offline` имеют статус:

```
Rejected
```

* одна запись удалена.
