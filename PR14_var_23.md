## Вариант 23. Проверка юридических лиц

### 📖 Описание ситуации

В юридический отдел поступают заявки на проверку контрагентов. Каждая заявка представляет собой ZIP-архив, содержащий три PDF-файла (все с текстовым слоем):

- `company.pdf` – наименование организации;
- `inn.pdf` – номер ИНН (10 или 12 цифр);
- `kpp.pdf` – номер КПП (9 цифр).

Робот должен:
1. Распаковать архив.
2. Извлечь текст из каждого PDF.
3. Проверить контрольную сумму ИНН.
4. Сформировать Excel-отчёт с результатами проверки.
5. Переместить архив в папку «Успех» или «Ошибки» в зависимости от результата.

### 🎯 Цель работы

Научиться применять локальные блоки Puzzle RPA для работы с PDF и проверки контрольных сумм:

- `Разархивировать файлы`
- `Считать текст из PDF файла` (или `Извлечь данные из PDF`)
- `Проверить контрольную сумму ИНН`
- `Записать в Excel`
- `Переместить файл/папку`
- `Создать папку`, `Существует путь`, `Список файлов`

### 📥 Входные данные

На диске в папке `C:\Incoming_Legal` находятся ZIP-архивы с именами вида `legal_*.zip`. Внутри каждого архива находятся три PDF-файла:

| Имя файла | Содержимое (текстовый слой) |
|-----------|------------------------------|
| `company.pdf` | `Наименование: ООО "Горизонт"` |
| `inn.pdf` | `ИНН: 7725397028` |
| `kpp.pdf` | `КПП: 772501001` |

**Важно:** все PDF создаются с текстовым слоем (не сканы). Студент самостоятельно готовит тестовые PDF (инструкция ниже).

### 📋 Задание

#### Этап 0. Подготовка рабочей среды (один раз)

0.1. Создать структуру папок:

C:\Incoming_Legal\ - папка для входящих ZIP-архивов

C:\Archived_Success\ - папка для успешно обработанных

C:\Archived_Errors\ - папка для заявок с ошибками

C:\Reports\ - папка для итоговых отчётов

C:\Temp\Extracted\ - временная папка для распаковки

0.2. Создать тестовые PDF-файлы согласно инструкции в разделе [«Данные для подготовки»](#-данные-для-подготовки-создаются-студентом-перед-выполнением).

#### Этап 1. Подготовка и проверка среды

1.1. Проверить существование папки `C:\Incoming_Legal`. Если нет – создать.

1.2. Получить список всех ZIP-файлов в `C:\Incoming_Legal` (блок «Список файлов/папок», маска `legal_*.zip`). Сохранить в переменную `zip_list`.

1.3. Если список пуст – вывести сообщение «Нет заявок» и завершить робота.

#### Этап 2. Цикл обработки каждой заявки

Для каждого ZIP-файла из `zip_list`:

2.1. **Разархивировать** во временную папку `C:\Temp\Extracted\[имя архива без .zip]`.

2.2. **Проверить наличие файлов** `company.pdf`, `inn.pdf`, `kpp.pdf`. Если какого-то нет – переместить ZIP в `Archived_Errors` и перейти к следующему.

2.3. **Извлечь данные из company.pdf**:
   - Использовать блок «Считать текст из PDF файла».
   - Из полученного текста извлечь наименование организации.

2.4. **Извлечь и проверить ИНН** из `inn.pdf`:
   - Считать текст, найти цифры (10 или 12) после слова «ИНН».
   - Передать в блок «Проверить контрольную сумму ИНН». Результат сохранить в `inn_valid`.

2.5. **Извлечь данные из kpp.pdf**:
   - Считать текст, найти КПП (9 цифр) для включения в отчёт.

2.6. **Принять решение**:
   - Если `inn_valid = True` → статус «Успех».
   - Иначе → статус «Ошибка».

#### Этап 3. Формирование Excel-отчёта

3.1. Если файл `C:\Reports\verification.xlsx` не существует – создать его с заголовками:  
   `Имя архива`, `Наименование компании`, `ИНН`, `ИНН валиден`, `КПП`, `Статус`.

3.2. Добавить строку с результатами текущей заявки.

#### Этап 4. Завершающие операции

4.1. Переместить ZIP-архив в папку `Archived_Success` или `Archived_Errors` в зависимости от статуса.

4.2. Удалить временную распакованную папку.

### 📊 Ожидаемый результат

- Excel-файл `C:\Reports\verification.xlsx` с результатами проверки всех заявок.
- ZIP-архивы рассортированы по папкам «Успех» и «Ошибки».

### 🗂 Данные для подготовки (создаются студентом перед выполнением)

#### Структура тестовых данных

Создать папку `C:\Incoming_Legal`. Внутри неё создать **три ZIP-архива** с именами:
1. `legal_20260421_100000.zip`
2. `legal_20260421_103000.zip`
3. `legal_20260421_110000.zip`

Каждый архив содержит три PDF-файла, созданных по инструкции ниже.

#### Инструкция по созданию PDF (текстовый слой)

1. **Создайте текстовые файлы в Word (или Google Docs, LibreOffice):**

   - Для `company.pdf` напишите:  
     `Наименование: ООО "Горизонт"`

   - Для `inn.pdf` напишите:  
     `ИНН: 7725397028`

   - Для `kpp.pdf` напишите:  
     `КПП: 772501001`

2. **Сохраните как PDF с текстовым слоем:**  
   В Microsoft Word: Файл → Сохранить как → PDF → Оптимизировать для стандартного (не минимальный размер).  
   Убедитесь, что в полученном PDF можно выделить текст мышью.

3. **Повторите для трёх заявок, изменив данные:**

   - **Заявка 2** (некорректный ИНН):  
     `inn.pdf` – `ИНН: 7725397029` (ошибка в контрольной сумме)

   - **Заявка 3** (корректный 12-значный ИНН):  
     `inn.pdf` – `ИНН: 123456789012`

4. **Заархивировать каждую заявку:**  
   - Создать отдельную папку для заявки, положить туда три PDF.  
   - Заархивировать папку (ПКМ → Отправить → Сжатая ZIP-папка).  
   - Переименовать ZIP-архив по шаблону и положить в `C:\Incoming_Legal`.
