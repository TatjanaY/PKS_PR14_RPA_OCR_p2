## Вариант 32. Проверка договоров

### 📖 Описание ситуации

В договорной отдел поступают заявки на проверку договоров. Каждая заявка представляет собой ZIP-архив (маска имени `doc_*`), содержащий три файла (два PDF с текстовым слоем + один CSV):

- `contract.pdf` – данные договора (номер, дата, стороны);
- `inn.pdf` – номер ИНН контрагента (10 или 12 цифр);
- `meta.csv` – метаданные заявки в формате CSV.

Робот должен:
1. Распаковать архив.
2. Извлечь текст из PDF-файлов и данные из CSV.
3. Проверить контрольную сумму ИНН.
4. Сформировать Excel-отчёт с базовыми колонками результатов.
5. Переместить архив в папку «Успех» или «Ошибки» в зависимости от результата (логика AND).

### 🎯 Цель работы

Научиться применять локальные блоки Puzzle RPA для работы с разными форматами данных и проверки ИНН:

- `Разархивировать файлы`
- `Считать текст из PDF файла`
- `Прочитать CSV файл`
- `Проверить контрольную сумму ИНН`
- `Записать в Excel`
- `Переместить файл/папку`

### 📥 Входные данные

На диске в папке `C:\Incoming_Contracts` находятся ZIP-архивы с именами вида `doc_*.zip`. Внутри каждого архива находятся файлы:

| Имя файла | Формат | Содержимое |
|-----------|--------|------------|
| `contract.pdf` | PDF (текстовый слой) | `Договор №Д-2026-001 от 2026-04-21; Стороны: ООО "Вектор" и ИП Иванов` |
| `inn.pdf` | PDF (текстовый слой) | `ИНН: 7725397028` |
| `meta.csv` | CSV | `id;date;manager\nDOC-321;2026-04-21;Petrova` |

**Важно:** все PDF создаются с текстовым слоем (не сканы). Студент самостоятельно готовит тестовые файлы.

### 📋 Задание

#### Этап 0. Подготовка рабочей среды (один раз)

0.1. Создать структуру папок:

C:\Incoming_Contracts\ - папка для входящих ZIP-архивов

C:\Archived_Success\ - папка для успешно обработанных

C:\Archived_Errors\ - папка для заявок с ошибками

C:\Reports\ - папка для итоговых отчётов

C:\Temp\Extracted\ - временная папка для распаковки

0.2. Создать тестовые файлы согласно инструкции в разделе [«Данные для подготовки»](#-данные-для-подготовки-создаются-студентом-перед-выполнением).

#### Этап 1. Подготовка и проверка среды

1.1. Проверить существование папки `C:\Incoming_Contracts`. Если нет – создать.

1.2. Получить список всех ZIP-файлов в `C:\Incoming_Contracts` (блок «Список файлов/папок», маска `doc_*.zip`). Сохранить в переменную `zip_list`.

1.3. Если список пуст – вывести сообщение «Нет заявок» и завершить робота.

#### Этап 2. Цикл обработки каждой заявки

Для каждого ZIP-файла из `zip_list`:

2.1. **Разархивировать** во временную папку `C:\Temp\Extracted\[имя архива без .zip]`.

2.2. **Проверить наличие файлов** `contract.pdf`, `inn.pdf`, `meta.csv`. Если какого-то нет – переместить ZIP в `Archived_Errors` и перейти к следующему.

2.3. **Извлечь данные из contract.pdf**:
   - Использовать блок «Считать текст из PDF файла».
   - Из полученного текста извлечь номер договора, дату и стороны.

2.4. **Извлечь и проверить ИНН** из `inn.pdf`:
   - Считать текст, найти цифры (10 или 12) после слова «ИНН».
   - Передать в блок «Проверить контрольную сумму ИНН». Результат сохранить в `inn_valid`.

2.5. **Прочитать метаданные из meta.csv**:
   - Использовать блок «Прочитать CSV файл».
   - Извлечь поля `id`, `date`, `manager` для возможного использования в отчёте.

2.6. **Принять решение (логика AND)**:
   - Если `inn_valid = True` → статус «Успех».
   - Иначе → статус «Ошибка».

#### Этап 3. Формирование Excel-отчёта

3.1. Если файл `C:\Reports\verification_v32.xlsx` не существует – создать его с базовыми заголовками:  
   `Имя архива`, `Номер договора`, `Дата договора`, `Стороны`, `ИНН`, `Валидность ИНН`, `ID заявки`, `Дата`, `Менеджер`, `Статус`.

3.2. Добавить строку с результатами текущей заявки.

#### Этап 4. Завершающие операции

4.1. Переместить ZIP-архив в папку `Archived_Success` или `Archived_Errors` в зависимости от статуса.

4.2. Удалить временную распакованную папку.

### 📊 Ожидаемый результат

- Excel-файл `C:\Reports\verification_v32.xlsx` с результатами проверки всех заявок.
- ZIP-архивы рассортированы по папкам «Успех» и «Ошибки». В «Успех» попадают только те, где прошла проверка ИНН.

### 🗂 Данные для подготовки (создаются студентом перед выполнением)

#### Структура тестовых данных

Создать папку `C:\Incoming_Contracts`. Внутри неё создать **три ZIP-архива** с именами:
1. `doc_001.zip`
2. `doc_002.zip`
3. `doc_003.zip`

Каждый архив содержит три файла, созданных по инструкции ниже.

#### Инструкция по созданию файлов

1. **Создайте текстовые файлы в Word (или аналогах) для PDF:**

   - Для `contract.pdf` напишите:  
     `Договор №Д-2026-001 от 2026-04-21; Стороны: ООО "Вектор" и ИП Иванов`

   - Для `inn.pdf` напишите:  
     `ИНН: 7725397028`

2. **Создайте CSV-файл meta.csv** (можно в Блокноте или Excel):
   ```
   id;date;manager
   DOC-321;2026-04-21;Petrova
   ```

3. **Сохраните как PDF с текстовым слоем:**  
   Файл → Сохранить как → PDF → Оптимизировать для стандартного. Убедитесь, что текст выделяется мышью.

4. **Повторите для трёх заявок, изменив данные:**

   - **Заявка 2** (ошибка в ИНН):  
     `inn.pdf` – `ИНН: 7725397029` (контрольная сумма не совпадает)

   - **Заявка 3** (корректный 12-значный ИНН):  
     `inn.pdf` – `ИНН: 123456789012` (валидный)

5. **Заархивировать каждую заявку:**  
   - Создать отдельную папку для заявки, положить туда два PDF и один CSV.  
   - Заархивировать папку (ПКМ → Отправить → Сжатая ZIP-папка).  
   - Переименовать ZIP-архив по шаблону (`doc_*.zip`) и положить в `C:\Incoming_Contracts`.
