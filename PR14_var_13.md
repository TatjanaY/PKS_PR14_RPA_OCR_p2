## Вариант 13. Проверка банковских реквизитов

### 📖 Описание ситуации

В финансовый отдел поступают заявки на проверку банковских реквизитов контрагентов. Каждая заявка представляет собой ZIP-архив, содержащий два PDF-файла (все с текстовым слоем):

- `bank_details.pdf` – банковские реквизиты (БИК, расчётный счёт, корреспондентский счёт);
- `client.pdf` – данные клиента (наименование организации, ИНН).

Робот должен:
1. Распаковать архив.
2. Извлечь текст из каждого PDF.
3. Проверить контрольную сумму расчётного счёта по БИК.
4. Сформировать Excel-отчёт с результатами проверки.
5. Переместить архив в папку «Успех» или «Ошибки» в зависимости от результата.

### 🎯 Цель работы

Научиться применять локальные блоки Puzzle RPA для работы с PDF и проверки контрольных сумм:

- `Разархивировать файлы`
- `Считать текст из PDF файла` (или `Извлечь данные из PDF`)
- `Проверить контрольную сумму Р/С или К/С`
- `Записать в Excel`
- `Переместить файл/папку`
- `Создать папку`, `Существует путь`, `Список файлов`

### 📥 Входные данные

На диске в папке `C:\Incoming_Bank` находятся ZIP-архивы с именами вида `bank_*.zip`. Внутри каждого архива находятся два PDF-файла:

| Имя файла | Содержимое (текстовый слой) |
|-----------|------------------------------|
| `bank_details.pdf` | `БИК=044525225; Р/С=40702810900000000001; К/С=30101810400000000225` |
| `client.pdf` | `Наименование: ООО "Ромашка"; ИНН: 7725397028` |

**Важно:** все PDF создаются с текстовым слоем (не сканы). Студент самостоятельно готовит тестовые PDF (инструкция ниже).

### 📋 Задание

#### Этап 0. Подготовка рабочей среды (один раз)

0.1. Создать структуру папок:

C:\Incoming_Bank\ - папка для входящих ZIP-архивов

C:\Archived_Success\ - папка для успешно обработанных

C:\Archived_Errors\ - папка для заявок с ошибками

C:\Reports\ - папка для итоговых отчётов

C:\Temp\Extracted\ - временная папка для распаковки

0.2. Создать тестовые PDF-файлы согласно инструкции в разделе [«Данные для подготовки»](#-данные-для-подготовки-создаются-студентом-перед-выполнением).

#### Этап 1. Подготовка и проверка среды

1.1. Проверить существование папки `C:\Incoming_Bank`. Если нет – создать.

1.2. Получить список всех ZIP-файлов в `C:\Incoming_Bank` (блок «Список файлов/папок», маска `bank_*.zip`). Сохранить в переменную `zip_list`.

1.3. Если список пуст – вывести сообщение «Нет заявок» и завершить робота.

#### Этап 2. Цикл обработки каждой заявки

Для каждого ZIP-файла из `zip_list`:

2.1. **Разархивировать** во временную папку `C:\Temp\Extracted\[имя архива без .zip]`.

2.2. **Проверить наличие файлов** `bank_details.pdf`, `client.pdf`. Если какого-то нет – переместить ZIP в `Archived_Errors` и перейти к следующему.

2.3. **Извлечь данные из bank_details.pdf**:
   - Использовать блок «Считать текст из PDF файла».
   - Из полученного текста извлечь БИК (9 цифр) и расчётный счёт (20 цифр) с помощью регулярного выражения.

2.4. **Проверить контрольную сумму расчётного счёта**:
   - Передать БИК и расчётный счёт в блок «Проверить контрольную сумму Р/С или К/С».
   - Результат сохранить в `account_valid`.

2.5. **Извлечь данные клиента из client.pdf** (для отчёта):
   - Считать текст, найти наименование организации и ИНН.

2.6. **Принять решение**:
   - Если `account_valid = True` → статус «Успех».
   - Иначе → статус «Ошибка».

#### Этап 3. Формирование Excel-отчёта

3.1. Если файл `C:\Reports\verification.xlsx` не существует – создать его с заголовками:  
   `Имя архива`, `Наименование клиента`, `ИНН`, `БИК`, `Расчётный счёт`, `Счёт валиден`, `Статус`.

3.2. Добавить строку с результатами текущей заявки.

#### Этап 4. Завершающие операции

4.1. Переместить ZIP-архив в папку `Archived_Success` или `Archived_Errors` в зависимости от статуса.

4.2. Удалить временную распакованную папку.

### 📊 Ожидаемый результат

- Excel-файл `C:\Reports\verification.xlsx` с результатами проверки всех заявок.
- ZIP-архивы рассортированы по папкам «Успех» и «Ошибки».

### 🗂 Данные для подготовки (создаются студентом перед выполнением)

#### Структура тестовых данных

Создать папку `C:\Incoming_Bank`. Внутри неё создать **три ZIP-архива** с именами:
1. `bank_20260421_100000.zip`
2. `bank_20260421_103000.zip`
3. `bank_20260421_110000.zip`

Каждый архив содержит два PDF-файла, созданных по инструкции ниже.

#### Инструкция по созданию PDF (текстовый слой)

1. **Создайте текстовые файлы в Word (или Google Docs, LibreOffice):**

   - Для `bank_details.pdf` напишите:  
     `БИК=044525225; Р/С=40702810900000000001; К/С=30101810400000000225`

   - Для `client.pdf` напишите:  
     `Наименование: ООО "Ромашка"; ИНН: 7725397028`

2. **Сохраните как PDF с текстовым слоем:**  
   В Microsoft Word: Файл → Сохранить как → PDF → Оптимизировать для стандартного (не минимальный размер).  
   Убедитесь, что в полученном PDF можно выделить текст мышью.

3. **Повторите для трёх заявок, изменив данные:**

   - **Заявка 2** (некорректный расчётный счёт):  
     `bank_details.pdf` – `БИК=044525225; Р/С=40702810900000000002` (ошибка в контрольной сумме)

   - **Заявка 3** (корректные реквизиты, другой банк):  
     `bank_details.pdf` – `БИК=044030134; Р/С=40702810500000000001` (контрольная сумма верна)

4. **Заархивировать каждую заявку:**  
   - Создать отдельную папку для заявки, положить туда два PDF.  
   - Заархивировать папку (ПКМ → Отправить → Сжатая ZIP-папка).  
   - Переименовать ZIP-архив по шаблону и положить в `C:\Incoming_Bank`.
