## Вариант 8. Проверка физических лиц

### 📖 Описание ситуации

В отдел по работе с физическими лицами поступают заявки на проверку данных граждан. Каждая заявка представляет собой ZIP-архив (маска имени `person_*`), содержащий четыре файла (три PDF с текстовым слоем + один CSV):

- `passport.pdf` – данные паспорта (фамилия, имя, отчество);
- `inn.pdf` – номер ИНН (10 или 12 цифр);
- `snils.pdf` – номер СНИЛС (11 цифр в формате `XXX-XXX-XXX-XX`);
- `meta.csv` – метаданные заявки в формате CSV.

Робот должен:
1. Распаковать архив.
2. Извлечь текст из PDF-файлов и данные из CSV.
3. Проверить контрольную сумму СНИЛС.
4. Сформировать Excel-отчёт с базовыми колонками результатов.
5. Переместить архив в папку «Успех» или «Ошибки» в зависимости от результата (логика AND).

### 🎯 Цель работы

Научиться применять локальные блоки Puzzle RPA для работы с разными форматами данных и проверки СНИЛС:

- `Разархивировать файлы`
- `Считать текст из PDF файла`
- `Прочитать CSV файл`
- `Проверить контрольную сумму СНИЛС`
- `Записать в Excel`
- `Переместить файл/папку`
- `Создать папку`, `Существует путь`, `Список файлов`

### 📥 Входные данные

На диске в папке `C:\Incoming_Persons` находятся ZIP-архивы с именами вида `person_*.zip`. Внутри каждого архива находятся файлы:

| Имя файла | Формат | Содержимое |
|-----------|--------|------------|
| `passport.pdf` | PDF (текстовый слой) | `Фамилия: Васильев; Имя: Павел; Отчество: Игоревич` |
| `inn.pdf` | PDF (текстовый слой) | `ИНН: 7725397028` |
| `snils.pdf` | PDF (текстовый слой) | `СНИЛС: 123-456-789-01` |
| `meta.csv` | CSV | `id;date;operator\n54321;2026-04-21;Sidorova` |

**Важно:** все PDF создаются с текстовым слоем (не сканы). Студент самостоятельно готовит тестовые файлы.

### 📋 Задание

#### Этап 0. Подготовка рабочей среды (один раз)

0.1. Создать структуру папок:

C:\Incoming_Persons\ - папка для входящих ZIP-архивов

C:\Archived_Success\ - папка для успешно обработанных

C:\Archived_Errors\ - папка для заявок с ошибками

C:\Reports\ - папка для итоговых отчётов

C:\Temp\Extracted\ - временная папка для распаковки

0.2. Создать тестовые файлы согласно инструкции в разделе [«Данные для подготовки»](#-данные-для-подготовки-создаются-студентом-перед-выполнением).

#### Этап 1. Подготовка и проверка среды

1.1. Проверить существование папки `C:\Incoming_Persons`. Если нет – создать.

1.2. Получить список всех ZIP-файлов в `C:\Incoming_Persons` (блок «Список файлов/папок», маска `person_*.zip`). Сохранить в переменную `zip_list`.

1.3. Если список пуст – вывести сообщение «Нет заявок» и завершить робота.

#### Этап 2. Цикл обработки каждой заявки

Для каждого ZIP-файла из `zip_list`:

2.1. **Разархивировать** во временную папку `C:\Temp\Extracted\[имя архива без .zip]`.

2.2. **Проверить наличие файлов** `passport.pdf`, `inn.pdf`, `snils.pdf`, `meta.csv`. Если какого-то нет – переместить ZIP в `Archived_Errors` и перейти к следующему.

2.3. **Извлечь данные из passport.pdf**:
   - Использовать блок «Считать текст из PDF файла».
   - Из полученного текста извлечь фамилию, имя, отчество (например, с помощью регулярных выражений).

2.4. **Извлечь и проверить СНИЛС** из `snils.pdf`:
   - Считать текст, найти номер СНИЛС (11 цифр, возможно с дефисами).
   - Передать в блок «Проверить контрольную сумму СНИЛС». Результат сохранить в `snils_valid`.

2.5. **Прочитать метаданные из meta.csv**:
   - Использовать блок «Прочитать CSV файл».
   - Извлечь поля `id`, `date`, `operator` для возможного использования в отчёте.

2.6. **Принять решение (логика AND)**:
   - Если `snils_valid = True` → статус «Успех».
   - Иначе → статус «Ошибка».

#### Этап 3. Формирование Excel-отчёта

3.1. Если файл `C:\Reports\verification_v8.xlsx` не существует – создать его с базовыми заголовками:  
   `Имя архива`, `Фамилия`, `Имя`, `Отчество`, `ИНН`, `СНИЛС`, `Валидность СНИЛС`, `ID заявки`, `Дата`, `Оператор`, `Статус`.

3.2. Добавить строку с результатами текущей заявки.

#### Этап 4. Завершающие операции

4.1. Переместить ZIP-архив в папку `Archived_Success` или `Archived_Errors` в зависимости от статуса.

4.2. Удалить временную распакованную папку.

### 📊 Ожидаемый результат

- Excel-файл `C:\Reports\verification_v8.xlsx` с результатами проверки всех заявок.
- ZIP-архивы рассортированы по папкам «Успех» и «Ошибки». В «Успех» попадают только те, где прошла проверка СНИЛС.

### 🗂 Данные для подготовки (создаются студентом перед выполнением)

#### Структура тестовых данных

Создать папку `C:\Incoming_Persons`. Внутри неё создать **три ZIP-архива** с именами:
1. `person_001.zip`
2. `person_002.zip`
3. `person_003.zip`

Каждый архив содержит четыре файла, созданных по инструкции ниже.

#### Инструкция по созданию файлов

1. **Создайте текстовые файлы в Word (или аналогах) для PDF:**

   - Для `passport.pdf` напишите:  
     `Фамилия: Васильев; Имя: Павел; Отчество: Игоревич`

   - Для `inn.pdf` напишите:  
     `ИНН: 7725397028`

   - Для `snils.pdf` напишите:  
     `СНИЛС: 123-456-789-01`

2. **Создайте CSV-файл meta.csv** (можно в Блокноте или Excel):
   ```
   id;date;operator
   54321;2026-04-21;Sidorova
   ```

3. **Сохраните как PDF с текстовым слоем:**  
   Файл → Сохранить как → PDF → Оптимизировать для стандартного. Убедитесь, что текст выделяется мышью.

4. **Повторите для трёх заявок, изменив данные:**

   - **Заявка 2** (ошибка в СНИЛС):  
     `snils.pdf` – `СНИЛС: 123-456-789-02` (контрольная сумма не совпадает)

   - **Заявка 3** (корректный СНИЛС):  
     `snils.pdf` – `СНИЛС: 987-654-321-09` (валидный)

5. **Заархивировать каждую заявку:**  
   - Создать отдельную папку для заявки, положить туда три PDF и один CSV.  
   - Заархивировать папку (ПКМ → Отправить → Сжатая ZIP-папка).  
   - Переименовать ZIP-архив по шаблону (`person_*.zip`) и положить в `C:\Incoming_Persons`.
