## Вариант 28. Проверка платёжных реквизитов

### 📖 Описание ситуации

В платёжный отдел поступают заявки на проверку платёжных реквизитов. Каждая заявка представляет собой ZIP-архив (маска имени `pay_*`), содержащий четыре файла (два PDF с текстовым слоем + один XML):

- `payment.pdf` – данные платежа (сумма, назначение);
- `bank.pdf` – банковские реквизиты (БИК, расчётный счёт);
- `meta.xml` – метаданные заявки в формате XML.

Робот должен:
1. Распаковать архив.
2. Извлечь текст из PDF-файлов и данные из XML.
3. Проверить контрольную сумму расчётного счёта по БИК.
4. Сформировать Excel-отчёт с базовыми колонками результатов.
5. Переместить архив в папку «Успех» или «Ошибки» в зависимости от результата (логика AND).

### 🎯 Цель работы

Научиться применять локальные блоки Puzzle RPA для работы с разными форматами данных и проверки банковских реквизитов:

- `Разархивировать файлы`
- `Считать текст из PDF файла`
- `Прочитать XML файл`
- `Проверить контрольную сумму Р/С или К/С`
- `Записать в Excel`
- `Переместить файл/папку`
- `Создать папку`, `Существует путь`, `Список файлов`

### 📥 Входные данные

На диске в папке `C:\Incoming_Payments` находятся ZIP-архивы с именами вида `pay_*.zip`. Внутри каждого архива находятся файлы:

| Имя файла | Формат | Содержимое |
|-----------|--------|------------|
| `payment.pdf` | PDF (текстовый слой) | `Сумма: 150000.00; Назначение: Оплата по договору №123` |
| `bank.pdf` | PDF (текстовый слой) | `БИК=044525225; Р/С=40702810900000000001` |
| `meta.xml` | XML | `<meta><id>PAY-281</id><date>2026-04-21</date><operator>Fedorova</operator></meta>` |

**Важно:** все PDF создаются с текстовым слоем (не сканы). Студент самостоятельно готовит тестовые файлы.

### 📋 Задание

#### Этап 0. Подготовка рабочей среды (один раз)

0.1. Создать структуру папок:

C:\Incoming_Payments\ - папка для входящих ZIP-архивов

C:\Archived_Success\ - папка для успешно обработанных

C:\Archived_Errors\ - папка для заявок с ошибками

C:\Reports\ - папка для итоговых отчётов

C:\Temp\Extracted\ - временная папка для распаковки

0.2. Создать тестовые файлы согласно инструкции в разделе [«Данные для подготовки»](#-данные-для-подготовки-создаются-студентом-перед-выполнением).

#### Этап 1. Подготовка и проверка среды

1.1. Проверить существование папки `C:\Incoming_Payments`. Если нет – создать.

1.2. Получить список всех ZIP-файлов в `C:\Incoming_Payments` (блок «Список файлов/папок», маска `pay_*.zip`). Сохранить в переменную `zip_list`.

1.3. Если список пуст – вывести сообщение «Нет заявок» и завершить робота.

#### Этап 2. Цикл обработки каждой заявки

Для каждого ZIP-файла из `zip_list`:

2.1. **Разархивировать** во временную папку `C:\Temp\Extracted\[имя архива без .zip]`.

2.2. **Проверить наличие файлов** `payment.pdf`, `bank.pdf`, `meta.xml`. Если какого-то нет – переместить ZIP в `Archived_Errors` и перейти к следующему.

2.3. **Извлечь данные из payment.pdf**:
   - Использовать блок «Считать текст из PDF файла».
   - Из полученного текста извлечь сумму и назначение платежа.

2.4. **Извлечь и проверить банковские реквизиты** из `bank.pdf`:
   - Считать текст, найти БИК (9 цифр) и расчётный счёт (20 цифр).
   - Передать в блок «Проверить контрольную сумму Р/С или К/С». Результат сохранить в `account_valid`.

2.5. **Прочитать метаданные из meta.xml**:
   - Использовать блок «Прочитать XML файл» или парсинг текста.
   - Извлечь поля `id`, `date`, `operator` для возможного использования в отчёте.

2.6. **Принять решение (логика AND)**:
   - Если `account_valid = True` → статус «Успех».
   - Иначе → статус «Ошибка».

#### Этап 3. Формирование Excel-отчёта

3.1. Если файл `C:\Reports\verification_v28.xlsx` не существует – создать его с базовыми заголовками:  
   `Имя архива`, `Сумма`, `Назначение`, `БИК`, `Расчётный счёт`, `Валидность счёта`, `ID заявки`, `Дата`, `Оператор`, `Статус`.

3.2. Добавить строку с результатами текущей заявки.

#### Этап 4. Завершающие операции

4.1. Переместить ZIP-архив в папку `Archived_Success` или `Archived_Errors` в зависимости от статуса.

4.2. Удалить временную распакованную папку.

### 📊 Ожидаемый результат

- Excel-файл `C:\Reports\verification_v28.xlsx` с результатами проверки всех заявок.
- ZIP-архивы рассортированы по папкам «Успех» и «Ошибки». В «Успех» попадают только те, где прошла проверка расчётного счёта.

### 🗂 Данные для подготовки (создаются студентом перед выполнением)

#### Структура тестовых данных

Создать папку `C:\Incoming_Payments`. Внутри неё создать **три ZIP-архива** с именами:
1. `pay_001.zip`
2. `pay_002.zip`
3. `pay_003.zip`

Каждый архив содержит три файла, созданных по инструкции ниже.

#### Инструкция по созданию файлов

1. **Создайте текстовые файлы в Word (или аналогах) для PDF:**

   - Для `payment.pdf` напишите:  
     `Сумма: 150000.00; Назначение: Оплата по договору №123`

   - Для `bank.pdf` напишите:  
     `БИК=044525225; Р/С=40702810900000000001`

2. **Создайте XML-файл meta.xml** (можно в Блокноте):
   ```xml
   <?xml version="1.0" encoding="UTF-8"?>
   <meta>
     <id>PAY-281</id>
     <date>2026-04-21</date>
     <operator>Fedorova</operator>
   </meta>
   ```

3. **Сохраните как PDF с текстовым слоем:**  
   Файл → Сохранить как → PDF → Оптимизировать для стандартного. Убедитесь, что текст выделяется мышью.

4. **Повторите для трёх заявок, изменив данные:**

   - **Заявка 2** (ошибка в счёте):  
     `bank.pdf` – `БИК=044525225; Р/С=40702810900000000002` (контрольная сумма не совпадает)

   - **Заявка 3** (корректные реквизиты):  
     `bank.pdf` – `БИК=044525225; Р/С=40702810900000000001` (валидный)

5. **Заархивировать каждую заявку:**  
   - Создать отдельную папку для заявки, положить туда два PDF и один XML.  
   - Заархивировать папку (ПКМ → Отправить → Сжатая ZIP-папка).  
   - Переименовать ZIP-архив по шаблону (`pay_*.zip`) и положить в `C:\Incoming_Payments`.
