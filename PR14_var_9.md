## Вариант 9. Проверка физических лиц (паспорт+ИНН+СНИЛС+JSON)

### 📖 Описание ситуации

В отдел кадров поступают заявки на проверку физических лиц. Каждая заявка представляет собой ZIP-архив, содержащий четыре файла (все с текстовым слоем для PDF):

- `passport.pdf` – данные паспорта (фамилия, имя, отчество, серия, номер);
- `inn.pdf` – номер ИНН (10 или 12 цифр);
- `snils.pdf` – номер СНИЛС (11 цифр в формате `XXX-XXX-XXX-XX`);
- `meta.json` – метаданные заявки в формате JSON.

Робот должен:
1. Распаковать архив.
2. Извлечь текст из каждого PDF и распарсить JSON.
3. Проверить контрольную сумму СНИЛС.
4. Сформировать Excel-отчёт с результатами проверки.
5. Переместить архив в папку «Успех» или «Ошибки» в зависимости от результата.

### 🎯 Цель работы

Научиться применять локальные блоки Puzzle RPA для работы с PDF, JSON и проверки контрольных сумм:

- `Разархивировать файлы`
- `Считать текст из PDF файла` (или `Извлечь данные из PDF`)
- `Прочитать JSON файл`
- `Проверить контрольную сумму СНИЛС`
- `Записать в Excel`
- `Переместить файл/папку`
- `Создать папку`, `Существует путь`, `Список файлов`

### 📥 Входные данные

На диске в папке `C:\Incoming_Persons` находятся ZIP-архивы с именами вида `person_*.zip`. Внутри каждого архива находятся четыре файла:

| Имя файла | Содержимое (текстовый слой / структура) |
|-----------|------------------------------|
| `passport.pdf` | `Фамилия: Сидоров; Имя: Сергей; Отчество: Сергеевич; Серия: 4502; Номер: 654321` |
| `inn.pdf` | `ИНН: 7725397028` |
| `snils.pdf` | `СНИЛС: 123-456-789-01` |
| `meta.json` | `{"request_id": "PERS-001", "timestamp": "2026-04-21T10:00:00", "source": "hr_portal"}` |

**Важно:** все PDF создаются с текстовым слоем (не сканы). JSON-файл должен быть валидным. Студент самостоятельно готовит тестовые файлы.

### 📋 Задание

#### Этап 0. Подготовка рабочей среды (один раз)

0.1. Создать структуру папок:

C:\Incoming_Persons\ - папка для входящих ZIP-архивов

C:\Archived_Success\ - папка для успешно обработанных

C:\Archived_Errors\ - папка для заявок с ошибками

C:\Reports\ - папка для итоговых отчётов

C:\Temp\Extracted\ - временная папка для распаковки

0.2. Создать тестовые файлы согласно инструкции в разделе [«Данные для подготовки»](#-данные-для-подготовки-создаются-студентом-перед-выполнением).

#### Этап 1. Подготовка и проверка среды

1.1. Проверить существование папки `C:\Incoming_Persons`. Если нет – создать.

1.2. Получить список всех ZIP-файлов в `C:\Incoming_Persons` (блок «Список файлов/папок», маска `person_*.zip`). Сохранить в переменную `zip_list`.

1.3. Если список пуст – вывести сообщение «Нет заявок» и завершить робота.

#### Этап 2. Цикл обработки каждой заявки

Для каждого ZIP-файла из `zip_list`:

2.1. **Разархивировать** во временную папку `C:\Temp\Extracted\[имя архива без .zip]`.

2.2. **Проверить наличие файлов** `passport.pdf`, `inn.pdf`, `snils.pdf`, `meta.json`. Если какого-то нет – переместить ZIP в `Archived_Errors` и перейти к следующему.

2.3. **Извлечь данные из passport.pdf**:
   - Использовать блок «Считать текст из PDF файла».
   - Из полученного текста извлечь фамилию, имя, отчество (остальные поля паспорта не требуются).

2.4. **Извлечь и проверить СНИЛС** из `snils.pdf`:
   - Считать текст, найти 11 цифр (в формате `XXX-XXX-XXX-XX` или сплошной строкой).
   - Передать в блок «Проверить контрольную сумму СНИЛС». Результат сохранить в `snils_valid`.

2.5. **Прочитать метаданные из meta.json**:
   - Использовать блок «Прочитать JSON файл».
   - Извлечь поля `request_id`, `timestamp`, `source` (для логирования, не для валидации).

2.6. **Принять решение (логика AND)**:
   - Если `snils_valid = True` → статус «Успех».
   - Иначе → статус «Ошибка».

#### Этап 3. Формирование Excel-отчёта

3.1. Если файл `C:\Reports\verification.xlsx` не существует – создать его с базовыми заголовками:  
   `Имя архива`, `Фамилия`, `Имя`, `Отчество`, `СНИЛС`, `СНИЛС валиден`, `Request ID`, `Статус`.

3.2. Добавить строку с результатами текущей заявки.

#### Этап 4. Завершающие операции

4.1. Переместить ZIP-архив в папку `Archived_Success` или `Archived_Errors` в зависимости от статуса.

4.2. Удалить временную распакованную папку.

### 📊 Ожидаемый результат

- Excel-файл `C:\Reports\verification.xlsx` с результатами проверки всех заявок (базовый набор колонок).
- ZIP-архивы рассортированы по папкам «Успех» и «Ошибки».

### 🗂 Данные для подготовки (создаются студентом перед выполнением)

#### Структура тестовых данных

Создать папку `C:\Incoming_Persons`. Внутри неё создать **три ZIP-архива** с именами:
1. `person_20260421_100000.zip`
2. `person_20260421_103000.zip`
3. `person_20260421_110000.zip`

Каждый архив содержит четыре файла, созданных по инструкции ниже.

#### Инструкция по созданию файлов (текстовый слой для PDF, валидный JSON)

1. **Создайте текстовые файлы в Word (или Google Docs, LibreOffice):**

   - Для `passport.pdf` напишите:  
     `Фамилия: Сидоров; Имя: Сергей; Отчество: Сергеевич; Серия: 4502; Номер: 654321`

   - Для `inn.pdf` напишите:  
     `ИНН: 7725397028`

   - Для `snils.pdf` напишите:  
     `СНИЛС: 123-456-789-01`

2. **Создайте JSON-файл** в текстовом редакторе:

   - Для `meta.json` напишите:  
     `{"request_id": "PERS-001", "timestamp": "2026-04-21T10:00:00", "source": "hr_portal"}`

3. **Сохраните как PDF с текстовым слоем:**  
   В Microsoft Word: Файл → Сохранить как → PDF → Оптимизировать для стандартного (не минимальный размер).  
   Убедитесь, что в полученном PDF можно выделить текст мышью.

4. **Повторите для трёх заявок, изменив данные:**

   - **Заявка 2** (некорректный СНИЛС):  
     `snils.pdf` – `СНИЛС: 123-456-789-02` (ошибка в контрольной сумме)  
     `meta.json` – `{"request_id": "PERS-002", "timestamp": "2026-04-21T10:30:00", "source": "api"}`

   - **Заявка 3** (корректный СНИЛС, другой формат):  
     `snils.pdf` – `СНИЛС: 98765432109` (сплошной формат, контрольная сумма верна)  
     `meta.json` – `{"request_id": "PERS-003", "timestamp": "2026-04-21T11:00:00", "source": "mobile"}`

5. **Заархивировать каждую заявку:**  
   - Создать отдельную папку для заявки, положить туда три PDF и один JSON.  
   - Заархивировать папку (ПКМ → Отправить → Сжатая ZIP-папка).  
   - Переименовать ZIP-архив по шаблону и положить в `C:\Incoming_Persons`.
