## Вариант 14. Проверка банковских реквизитов

### 📖 Описание ситуации

В банковский отдел поступают заявки на проверку реквизитов клиентов. Каждая заявка представляет собой ZIP-архив (маска имени `bank_*`), содержащий три файла (два PDF с текстовым слоем + один CSV):

- `bank_details.pdf` – банковские реквизиты (БИК, расчётный счёт);
- `client.pdf` – данные клиента (наименование, ИНН);
- `meta.csv` – метаданные заявки в формате CSV.

Робот должен:
1. Распаковать архив.
2. Извлечь текст из PDF-файлов и данные из CSV.
3. Проверить контрольную сумму расчётного счёта по БИК.
4. Сформировать Excel-отчёт с базовыми колонками результатов.
5. Переместить архив в папку «Успех» или «Ошибки» в зависимости от результата (логика AND).

### 🎯 Цель работы

Научиться применять локальные блоки Puzzle RPA для работы с разными форматами данных и проверки банковских реквизитов:

- `Разархивировать файлы`
- `Считать текст из PDF файла`
- `Прочитать CSV файл`
- `Проверить контрольную сумму Р/С или К/С`
- `Записать в Excel`
- `Переместить файл/папку`
- `Создать папку`, `Существует путь`, `Список файлов`

### 📥 Входные данные

На диске в папке `C:\Incoming_Bank` находятся ZIP-архивы с именами вида `bank_*.zip`. Внутри каждого архива находятся файлы:

| Имя файла | Формат | Содержимое |
|-----------|--------|------------|
| `bank_details.pdf` | PDF (текстовый слой) | `БИК=044525225; Р/С=40702810900000000001` |
| `client.pdf` | PDF (текстовый слой) | `Клиент: ООО "Вектор"; ИНН: 7725397028` |
| `meta.csv` | CSV | `id;date;manager\nBNK-101;2026-04-21;Volkova` |

**Важно:** все PDF создаются с текстовым слоем (не сканы). Студент самостоятельно готовит тестовые файлы.

### 📋 Задание

#### Этап 0. Подготовка рабочей среды (один раз)

0.1. Создать структуру папок:

C:\Incoming_Bank\ - папка для входящих ZIP-архивов

C:\Archived_Success\ - папка для успешно обработанных

C:\Archived_Errors\ - папка для заявок с ошибками

C:\Reports\ - папка для итоговых отчётов

C:\Temp\Extracted\ - временная папка для распаковки

0.2. Создать тестовые файлы согласно инструкции в разделе [«Данные для подготовки»](#-данные-для-подготовки-создаются-студентом-перед-выполнением).

#### Этап 1. Подготовка и проверка среды

1.1. Проверить существование папки `C:\Incoming_Bank`. Если нет – создать.

1.2. Получить список всех ZIP-файлов в `C:\Incoming_Bank` (блок «Список файлов/папок», маска `bank_*.zip`). Сохранить в переменную `zip_list`.

1.3. Если список пуст – вывести сообщение «Нет заявок» и завершить робота.

#### Этап 2. Цикл обработки каждой заявки

Для каждого ZIP-файла из `zip_list`:

2.1. **Разархивировать** во временную папку `C:\Temp\Extracted\[имя архива без .zip]`.

2.2. **Проверить наличие файлов** `bank_details.pdf`, `client.pdf`, `meta.csv`. Если какого-то нет – переместить ZIP в `Archived_Errors` и перейти к следующему.

2.3. **Извлечь данные из bank_details.pdf**:
   - Использовать блок «Считать текст из PDF файла».
   - Из полученного текста извлечь БИК (9 цифр) и расчётный счёт (20 цифр).

2.4. **Проверить расчётный счёт** из `bank_details.pdf`:
   - Передать БИК и счёт в блок «Проверить контрольную сумму Р/С или К/С». Результат сохранить в `account_valid`.

2.5. **Извлечь данные клиента из client.pdf**:
   - Считать текст, найти наименование клиента и ИНН.

2.6. **Прочитать метаданные из meta.csv**:
   - Использовать блок «Прочитать CSV файл».
   - Извлечь поля `id`, `date`, `manager` для возможного использования в отчёте.

2.7. **Принять решение (логика AND)**:
   - Если `account_valid = True` → статус «Успех».
   - Иначе → статус «Ошибка».

#### Этап 3. Формирование Excel-отчёта

3.1. Если файл `C:\Reports\verification_v14.xlsx` не существует – создать его с базовыми заголовками:  
   `Имя архива`, `Клиент`, `ИНН`, `БИК`, `Расчётный счёт`, `Валидность счёта`, `ID заявки`, `Дата`, `Менеджер`, `Статус`.

3.2. Добавить строку с результатами текущей заявки.

#### Этап 4. Завершающие операции

4.1. Переместить ZIP-архив в папку `Archived_Success` или `Archived_Errors` в зависимости от статуса.

4.2. Удалить временную распакованную папку.

### 📊 Ожидаемый результат

- Excel-файл `C:\Reports\verification_v14.xlsx` с результатами проверки всех заявок.
- ZIP-архивы рассортированы по папкам «Успех» и «Ошибки». В «Успех» попадают только те, где прошла проверка расчётного счёта.

### 🗂 Данные для подготовки (создаются студентом перед выполнением)

#### Структура тестовых данных

Создать папку `C:\Incoming_Bank`. Внутри неё создать **три ZIP-архива** с именами:
1. `bank_001.zip`
2. `bank_002.zip`
3. `bank_003.zip`

Каждый архив содержит три файла, созданных по инструкции ниже.

#### Инструкция по созданию файлов

1. **Создайте текстовые файлы в Word (или аналогах) для PDF:**

   - Для `bank_details.pdf` напишите:  
     `БИК=044525225; Р/С=40702810900000000001`

   - Для `client.pdf` напишите:  
     `Клиент: ООО "Вектор"; ИНН: 7725397028`

2. **Создайте CSV-файл meta.csv** (можно в Блокноте или Excel):
   ```
   id;date;manager
   BNK-101;2026-04-21;Volkova
   ```

3. **Сохраните как PDF с текстовым слоем:**  
   Файл → Сохранить как → PDF → Оптимизировать для стандартного. Убедитесь, что текст выделяется мышью.

4. **Повторите для трёх заявок, изменив данные:**

   - **Заявка 2** (ошибка в счёте):  
     `bank_details.pdf` – `БИК=044525225; Р/С=40702810900000000002` (контрольная сумма не совпадает)

   - **Заявка 3** (корректные реквизиты):  
     `bank_details.pdf` – `БИК=044525225; Р/С=40702810900000000001` (валидный)

5. **Заархивировать каждую заявку:**  
   - Создать отдельную папку для заявки, положить туда два PDF и один CSV.  
   - Заархивировать папку (ПКМ → Отправить → Сжатая ZIP-папка).  
   - Переименовать ZIP-архив по шаблону (`bank_*.zip`) и положить в `C:\Incoming_Bank`.
