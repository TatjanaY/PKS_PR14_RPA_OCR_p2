## Вариант 36. Проверка договоров

### 📖 Описание ситуации

В договорной отдел поступают заявки на проверку договоров с контрагентами. Каждая заявка представляет собой ZIP-архив, содержащий четыре файла в смешанном формате:

- `contract.pdf` – данные договора (номер, дата, сумма, стороны);
- `inn.pdf` – номер ИНН контрагента (10 или 12 цифр);
- `info.txt` – дополнительные текстовые примечания к заявке;
- `meta.json` – метаданные заявки в формате JSON.

Робот должен:
1. Распаковать архив.
2. Извлечь текст из PDF, прочитать содержимое TXT и распарсить JSON.
3. Проверить контрольную сумму ИНН.
4. Сформировать расширенный Excel-отчёт с результатами проверки.
5. Переместить архив в папку «Успех» или «Ошибки» в зависимости от результата.

### 🎯 Цель работы

Научиться применять локальные блоки Puzzle RPA для работы с MIX-форматами и проверки контрольных сумм:

- `Разархивировать файлы`
- `Считать текст из PDF файла` (или `Извлечь данные из PDF`)
- `Прочитать TXT файл` (или `Считать текст из файла`)
- `Прочитать JSON файл`
- `Проверить контрольную сумму ИНН`
- `Записать в Excel`
- `Переместить файл/папку`
- `Создать папку`, `Существует путь`, `Список файлов`

### 📥 Входные данные

На диске в папке `C:\Incoming_Contracts` находятся ZIP-архивы с именами вида `doc_*.zip`. Внутри каждого архива находятся четыре файла:

| Имя файла | Содержимое (текстовый слой / структура) |
|-----------|------------------------------|
| `contract.pdf` | `Договор №789-ДГ от 25.03.2026; Сумма: 750000.00; Стороны: ООО "Гамма" и ООО "Дельта"` |
| `inn.pdf` | `ИНН: 7725397028` |
| `info.txt` | `Примечание: Срочная оплата. Контактное лицо: менеджер Иванов.` |
| `meta.json` | `{"request_id": "DOC-001", "timestamp": "2026-04-21T10:00:00", "priority": "high"}` |

**Важно:** все PDF создаются с текстовым слоем (не сканы). TXT и JSON должны быть валидными. Студент самостоятельно готовит тестовые файлы.

### 📋 Задание

#### Этап 0. Подготовка рабочей среды (один раз)

0.1. Создать структуру папок:

C:\Incoming_Contracts\ - папка для входящих ZIP-архивов

C:\Archived_Success\ - папка для успешно обработанных

C:\Archived_Errors\ - папка для заявок с ошибками

C:\Reports\ - папка для итоговых отчётов

C:\Temp\Extracted\ - временная папка для распаковки

0.2. Создать тестовые файлы согласно инструкции в разделе [«Данные для подготовки»](#-данные-для-подготовки-создаются-студентом-перед-выполнением).

#### Этап 1. Подготовка и проверка среды

1.1. Проверить существование папки `C:\Incoming_Contracts`. Если нет – создать.

1.2. Получить список всех ZIP-файлов в `C:\Incoming_Contracts` (блок «Список файлов/папок», маска `doc_*.zip`). Сохранить в переменную `zip_list`.

1.3. Если список пуст – вывести сообщение «Нет заявок» и завершить робота.

#### Этап 2. Цикл обработки каждой заявки

Для каждого ZIP-файла из `zip_list`:

2.1. **Разархивировать** во временную папку `C:\Temp\Extracted\[имя архива без .zip]`.

2.2. **Проверить наличие файлов** `contract.pdf`, `inn.pdf`, `info.txt`, `meta.json`. Если какого-то нет – переместить ZIP в `Archived_Errors` и перейти к следующему.

2.3. **Извлечь данные из contract.pdf**:
   - Использовать блок «Считать текст из PDF файла».
   - Из полученного текста извлечь все поля: номер договора, дату, сумму, стороны.

2.4. **Извлечь и проверить ИНН** из `inn.pdf`:
   - Считать текст, найти цифры (10 или 12) после слова «ИНН».
   - Передать в блок «Проверить контрольную сумму ИНН». Результат сохранить в `inn_valid`.

2.5. **Прочитать дополнительные данные**:
   - Прочитать содержимое `info.txt` в переменную `notes`.
   - Использовать блок «Прочитать JSON файл» для извлечения полей `request_id`, `timestamp`, `priority`.

2.6. **Принять решение (логика AND)**:
   - Если `inn_valid = True` → статус «Успех».
   - Иначе → статус «Ошибка».

#### Этап 3. Формирование Excel-отчёта

3.1. Если файл `C:\Reports\verification.xlsx` не существует – создать его с расширенными заголовками:  
   `Имя архива`, `Номер договора`, `Дата`, `Сумма`, `Стороны`, `ИНН`, `ИНН валиден`, `Примечание`, `Priority`, `Request ID`, `Статус`.

3.2. Добавить строку с результатами текущей заявки.

#### Этап 4. Завершающие операции

4.1. Переместить ZIP-архив в папку `Archived_Success` или `Archived_Errors` в зависимости от статуса.

4.2. Удалить временную распакованную папку.

### 📊 Ожидаемый результат

- Excel-файл `C:\Reports\verification.xlsx` с расширенным набором колонок и результатами проверки всех заявок.
- ZIP-архивы рассортированы по папкам «Успех» и «Ошибки».

### 🗂 Данные для подготовки (создаются студентом перед выполнением)

#### Структура тестовых данных

Создать папку `C:\Incoming_Contracts`. Внутри неё создать **три ZIP-архива** с именами:

#### Инструкция по созданию файлов (текстовый слой для PDF, валидный TXT и JSON)

1. **Создайте текстовые файлы в Word (или Google Docs, LibreOffice):**

   - Для `contract.pdf` напишите:  
     `Договор №789-ДГ от 25.03.2026; Сумма: 750000.00; Стороны: ООО "Гамма" и ООО "Дельта"`

   - Для `inn.pdf` напишите:  
     `ИНН: 7725397028`

2. **Создайте TXT-файл** в текстовом редакторе:

   - Для `info.txt` напишите:  
     `Примечание: Срочная оплата. Контактное лицо: менеджер Иванов.`

3. **Создайте JSON-файл** в текстовом редакторе:

   - Для `meta.json` напишите:  
     `{"request_id": "DOC-001", "timestamp": "2026-04-21T10:00:00", "priority": "high"}`

4. **Сохраните как PDF с текстовым слоем:**  
   В Microsoft Word: Файл → Сохранить как → PDF → Оптимизировать для стандартного (не минимальный размер).  
   Убедитесь, что в полученном PDF можно выделить текст мышью.

5. **Повторите для трёх заявок, изменив данные:**

   - **Заявка 2** (некорректный ИНН):  
     `inn.pdf` – `ИНН: 7725397029` (ошибка в контрольной сумме)  
     `info.txt` – `Примечание: Стандартная обработка.`  
     `meta.json` – `{"request_id": "DOC-002", "timestamp": "2026-04-21T10:30:00", "priority": "normal"}`

   - **Заявка 3** (корректный 12-значный ИНН):  
     `inn.pdf` – `ИНН: 123456789012`  
     `info.txt` – `Примечание: Требуется подпись директора.`  
     `meta.json` – `{"request_id": "DOC-003", "timestamp": "2026-04-21T11:00:00", "priority": "low"}`

6. **Заархивировать каждую заявку:**  
   - Создать отдельную папку для заявки, положить туда два PDF, один TXT и один JSON.  
   - Заархивировать папку (ПКМ → Отправить → Сжатая ZIP-папка).  
   - Переименовать ZIP-архив по шаблону и положить в `C:\Incoming_Contracts`.
