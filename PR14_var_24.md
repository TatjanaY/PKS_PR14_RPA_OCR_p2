## Вариант 24. Проверка юридических лиц

### 📖 Описание ситуации

В юридический отдел поступают заявки на проверку данных организаций. Каждая заявка представляет собой ZIP-архив (маска имени `legal_*`), содержащий пять файлов разных форматов:

- `company.pdf` – данные компании (наименование, ОГРН);
- `inn.pdf` – номер ИНН (10 или 12 цифр);
- `kpp.pdf` – код КПП (9 цифр);
- `notes.txt` – текстовые заметки юриста;
- `meta.json` – метаданные заявки в формате JSON.

Робот должен:
1. Распаковать архив.
2. Извлечь текст из PDF-файлов, прочитать TXT и JSON.
3. Проверить контрольную сумму ИНН.
4. Сформировать Excel-отчёт с расширенным набором колонок.
5. Переместить архив в папку «Успех» или «Ошибки» в зависимости от результата (логика AND).

### 🎯 Цель работы

Научиться применять локальные блоки Puzzle RPA для работы с разными форматами данных и проверки ИНН:

- `Разархивировать файлы`
- `Считать текст из PDF файла`
- `Прочитать текстовый файл`
- `Прочитать JSON файл`
- `Проверить контрольную сумму ИНН`
- `Записать в Excel`
- `Переместить файл/папку`
- `Создать папку`, `Существует путь`, `Список файлов`

### 📥 Входные данные

На диске в папке `C:\Incoming_Legal` находятся ZIP-архивы с именами вида `legal_*.zip`. Внутри каждого архива находятся файлы:

| Имя файла | Формат | Содержимое |
|-----------|--------|------------|
| `company.pdf` | PDF (текстовый слой) | `Наименование: ООО "СтройТрест"; ОГРН: 1027700123456` |
| `inn.pdf` | PDF (текстовый слой) | `ИНН: 7725397028` |
| `kpp.pdf` | PDF (текстовый слой) | `КПП: 772501001` |
| `notes.txt` | TXT | `Юрист: Кузнецов. Комментарий: Документы в порядке.` |
| `meta.json` | JSON | `{"id": "LGL-301", "date": "2026-04-21", "priority": "normal"}` |

**Важно:** все PDF создаются с текстовым слоем (не сканы). Студент самостоятельно готовит тестовые файлы.

### 📋 Задание

#### Этап 0. Подготовка рабочей среды (один раз)

0.1. Создать структуру папок:

C:\Incoming_Legal\ - папка для входящих ZIP-архивов

C:\Archived_Success\ - папка для успешно обработанных

C:\Archived_Errors\ - папка для заявок с ошибками

C:\Reports\ - папка для итоговых отчётов

C:\Temp\Extracted\ - временная папка для распаковки

0.2. Создать тестовые файлы согласно инструкции в разделе [«Данные для подготовки»](#-данные-для-подготовки-создаются-студентом-перед-выполнением).

#### Этап 1. Подготовка и проверка среды

1.1. Проверить существование папки `C:\Incoming_Legal`. Если нет – создать.

1.2. Получить список всех ZIP-файлов в `C:\Incoming_Legal` (блок «Список файлов/папок», маска `legal_*.zip`). Сохранить в переменную `zip_list`.

1.3. Если список пуст – вывести сообщение «Нет заявок» и завершить робота.

#### Этап 2. Цикл обработки каждой заявки

Для каждого ZIP-файла из `zip_list`:

2.1. **Разархивировать** во временную папку `C:\Temp\Extracted\[имя архива без .zip]`.

2.2. **Проверить наличие файлов** `company.pdf`, `inn.pdf`, `kpp.pdf`, `notes.txt`, `meta.json`. Если какого-то нет – переместить ZIP в `Archived_Errors` и перейти к следующему.

2.3. **Извлечь данные из company.pdf**:
   - Использовать блок «Считать текст из PDF файла».
   - Из полученного текста извлечь наименование компании и ОГРН.

2.4. **Извлечь и проверить ИНН** из `inn.pdf`:
   - Считать текст, найти цифры (10 или 12) после слова «ИНН».
   - Передать в блок «Проверить контрольную сумму ИНН». Результат сохранить в `inn_valid`.

2.5. **Извлечь данные из kpp.pdf**:
   - Считать текст, найти код КПП.

2.6. **Прочитать дополнительные данные**:
   - Из `notes.txt` извлечь комментарий юриста.
   - Из `meta.json` извлечь поля `id`, `date`, `priority`.

2.7. **Принять решение (логика AND)**:
   - Если `inn_valid = True` → статус «Успех».
   - Иначе → статус «Ошибка».

#### Этап 3. Формирование Excel-отчёта

3.1. Если файл `C:\Reports\verification_v24.xlsx` не существует – создать его с расширенными заголовками:  
   `Имя архива`, `Наименование`, `ОГРН`, `ИНН`, `Валидность ИНН`, `КПП`, `ID заявки`, `Дата`, `Приоритет`, `Комментарий`, `Статус`.

3.2. Добавить строку с результатами текущей заявки.

#### Этап 4. Завершающие операции

4.1. Переместить ZIP-архив в папку `Archived_Success` или `Archived_Errors` в зависимости от статуса.

4.2. Удалить временную распакованную папку.

### 📊 Ожидаемый результат

- Excel-файл `C:\Reports\verification_v24.xlsx` с расширенным набором колонок по всем заявкам.
- ZIP-архивы рассортированы по папкам «Успех» и «Ошибки». В «Успех» попадают только те, где прошла проверка ИНН.

### 🗂 Данные для подготовки (создаются студентом перед выполнением)

#### Структура тестовых данных

Создать папку `C:\Incoming_Legal`. Внутри неё создать **три ZIP-архива** с именами:
1. `legal_001.zip`
2. `legal_002.zip`
3. `legal_003.zip`

Каждый архив содержит пять файлов, созданных по инструкции ниже.

#### Инструкция по созданию файлов

1. **Создайте текстовые файлы в Word (или аналогах) для PDF:**

   - Для `company.pdf` напишите:  
     `Наименование: ООО "Альянс"; ОГРН: 1027700987654`

   - Для `inn.pdf` напишите:  
     `ИНН: 7725397028`

   - Для `kpp.pdf` напишите:  
     `КПП: 772501001`

2. **Создайте TXT-файл notes.txt** (Блокнот):
   ```
   Юрист: Кузнецов. Комментарий: Документы в порядке.
   ```

3. **Создайте JSON-файл meta.json** (Блокнот):
   ```json
   {
     "id": "LGL-301",
     "date": "2026-04-21",
     "priority": "normal"
   }
   ```

4. **Сохраните как PDF с текстовым слоем:**  
   Файл → Сохранить как → PDF → Оптимизировать для стандартного. Убедитесь, что текст выделяется мышью.

5. **Повторите для трёх заявок, изменив данные:**

   - **Заявка 2** (ошибка в ИНН):  
     `inn.pdf` – `ИНН: 7725397029` (контрольная сумма не совпадает)

   - **Заявка 3** (корректный 12-значный ИНН):  
     `inn.pdf` – `ИНН: 123456789012` (валидный)

6. **Заархивировать каждую заявку:**  
   - Создать отдельную папку для заявки, положить туда три PDF, TXT и JSON.  
   - Заархивировать папку (ПКМ → Отправить → Сжатая ZIP-папка).  
   - Переименовать ZIP-архив по шаблону (`legal_*.zip`) и положить в `C:\Incoming_Legal`.
