## Вариант 6. Верификация контрагентов

### 📖 Описание ситуации

В бухгалтерию поступают заявки на верификацию контрагентов. Каждая заявка представляет собой ZIP-архив (маска имени `contract_*`), содержащий пять файлов разных форматов:

- `passport.pdf` – данные паспорта (фамилия, имя, отчество, серия, номер);
- `inn.pdf` – номер ИНН (10 или 12 цифр);
- `bank.pdf` – БИК и расчётный счёт;
- `notes.txt` – текстовые заметки менеджера;
- `meta.json` – метаданные заявки в формате JSON.

Робот должен:
1. Распаковать архив.
2. Извлечь текст из PDF-файлов, прочитать TXT и JSON.
3. Проверить контрольную сумму ИНН.
4. Проверить контрольную сумму расчётного счёта по БИК.
5. Сформировать Excel-отчёт с расширенным набором колонок.
6. Переместить архив в папку «Успех» или «Ошибки» в зависимости от результата (логика AND).

### 🎯 Цель работы

Научиться применять локальные блоки Puzzle RPA для работы с разными форматами данных:

- `Разархивировать файлы`
- `Считать текст из PDF файла`
- `Прочитать текстовый файл`
- `Прочитать JSON файл`
- `Проверить контрольную сумму ИНН`
- `Проверить контрольную сумму Р/С или К/С`
- `Записать в Excel`
- `Переместить файл/папку`
- `Создать папку`, `Существует путь`, `Список файлов`

### 📥 Входные данные

На диске в папке `C:\Incoming_Contracts` находятся ZIP-архивы с именами вида `contract_*.zip`. Внутри каждого архива находятся файлы:

| Имя файла | Формат | Содержимое |
|-----------|--------|------------|
| `passport.pdf` | PDF (текстовый слой) | `Фамилия: Смирнов; Имя: Андрей; Отчество: Сергеевич; Серия: 4510; Номер: 112233` |
| `inn.pdf` | PDF (текстовый слой) | `ИНН: 7725397028` |
| `bank.pdf` | PDF (текстовый слой) | `БИК=044525225; Р/С=40702810900000000001` |
| `notes.txt` | TXT | `Менеджер: Иванова. Комментарий: Клиент премиум-сегмент.` |
| `meta.json` | JSON | `{"id": "REQ-001", "date": "2026-04-21", "priority": "high"}` |

**Важно:** все PDF создаются с текстовым слоем (не сканы). Студент самостоятельно готовит тестовые файлы.

### 📋 Задание

#### Этап 0. Подготовка рабочей среды (один раз)

0.1. Создать структуру папок:

C:\Incoming_Contracts\ - папка для входящих ZIP-архивов

C:\Archived_Success\ - папка для успешно обработанных

C:\Archived_Errors\ - папка для заявок с ошибками

C:\Reports\ - папка для итоговых отчётов

C:\Temp\Extracted\ - временная папка для распаковки

0.2. Создать тестовые файлы согласно инструкции в разделе [«Данные для подготовки»](#-данные-для-подготовки-создаются-студентом-перед-выполнением).

#### Этап 1. Подготовка и проверка среды

1.1. Проверить существование папки `C:\Incoming_Contracts`. Если нет – создать.

1.2. Получить список всех ZIP-файлов в `C:\Incoming_Contracts` (блок «Список файлов/папок», маска `contract_*.zip`). Сохранить в переменную `zip_list`.

1.3. Если список пуст – вывести сообщение «Нет заявок» и завершить робота.

#### Этап 2. Цикл обработки каждой заявки

Для каждого ZIP-файла из `zip_list`:

2.1. **Разархивировать** во временную папку `C:\Temp\Extracted\[имя архива без .zip]`.

2.2. **Проверить наличие файлов** `passport.pdf`, `inn.pdf`, `bank.pdf`, `notes.txt`, `meta.json`. Если какого-то нет – переместить ZIP в `Archived_Errors` и перейти к следующему.

2.3. **Извлечь данные из passport.pdf**:
   - Использовать блок «Считать текст из PDF файла».
   - Из полученного текста извлечь фамилию, имя, отчество, серию, номер (например, с помощью регулярных выражений).

2.4. **Извлечь и проверить ИНН** из `inn.pdf`:
   - Считать текст, найти цифры (10 или 12) после слова «ИНН».
   - Передать в блок «Проверить контрольную сумму ИНН». Результат сохранить в `inn_valid`.

2.5. **Извлечь и проверить банковские реквизиты** из `bank.pdf`:
   - Считать текст, найти БИК (9 цифр) и расчётный счёт (20 цифр).
   - Передать в блок «Проверить контрольную сумму Р/С или К/С». Результат сохранить в `account_valid`.

2.6. **Прочитать дополнительные данные**:
   - Из `notes.txt` извлечь комментарий менеджера.
   - Из `meta.json` извлечь поля `id`, `date`, `priority`.

2.7. **Принять решение (логика AND)**:
   - Если `inn_valid = True` и `account_valid = True` → статус «Успех».
   - Иначе → статус «Ошибка».

#### Этап 3. Формирование Excel-отчёта

3.1. Если файл `C:\Reports\verification_v6.xlsx` не существует – создать его с расширенными заголовками:  
   `Имя архива`, `Фамилия`, `Имя`, `Отчество`, `Серия`, `Номер паспорта`, `ИНН`, `Валидность ИНН`, `БИК`, `Расчётный счёт`, `Валидность счёта`, `ID заявки`, `Дата`, `Приоритет`, `Комментарий`, `Статус`.

3.2. Добавить строку с результатами текущей заявки.

#### Этап 4. Завершающие операции

4.1. Переместить ZIP-архив в папку `Archived_Success` или `Archived_Errors` в зависимости от статуса.

4.2. Удалить временную распакованную папку.

### 📊 Ожидаемый результат

- Excel-файл `C:\Reports\verification_v6.xlsx` с расширенным набором колонок по всем заявкам.
- ZIP-архивы рассортированы по папкам «Успех» и «Ошибки». В «Успех» попадают только те, где прошли **обе** проверки (ИНН и счёт).

### 🗂 Данные для подготовки (создаются студентом перед выполнением)

#### Структура тестовых данных

Создать папку `C:\Incoming_Contracts`. Внутри неё создать **три ZIP-архива** с именами:
1. `contract_001.zip`
2. `contract_002.zip`
3. `contract_003.zip`

Каждый архив содержит пять файлов, созданных по инструкции ниже.

#### Инструкция по созданию файлов

1. **Создайте текстовые файлы в Word (или аналогах) для PDF:**

   - Для `passport.pdf` напишите:  
     `Фамилия: Смирнов; Имя: Андрей; Отчество: Сергеевич; Серия: 4510; Номер: 112233`

   - Для `inn.pdf` напишите:  
     `ИНН: 7725397028`

   - Для `bank.pdf` напишите:  
     `БИК=044525225; Р/С=40702810900000000001`

2. **Создайте TXT-файл notes.txt** (Блокнот):
   ```
   Менеджер: Иванова. Комментарий: Клиент премиум-сегмент.
   ```

3. **Создайте JSON-файл meta.json** (Блокнот):
   ```json
   {
     "id": "REQ-001",
     "date": "2026-04-21",
     "priority": "high"
   }
   ```

4. **Сохраните как PDF с текстовым слоем:**  
   Файл → Сохранить как → PDF → Оптимизировать для стандартного. Убедитесь, что текст выделяется мышью.

5. **Повторите для трёх заявок, изменив данные:**

   - **Заявка 2** (ошибка в ИНН):  
     `inn.pdf` – `ИНН: 7725397029` (контрольная сумма не совпадает)

   - **Заявка 3** (ошибка в счёте):  
     `bank.pdf` – `БИК=044525225; Р/С=40702810900000000002` (неверный счёт)

6. **Заархивировать каждую заявку:**  
   - Создать отдельную папку для заявки, положить туда три PDF, TXT и JSON.  
   - Заархивировать папку (ПКМ → Отправить → Сжатая ZIP-папка).  
   - Переименовать ZIP-архив по шаблону (`contract_*.zip`) и положить в `C:\Incoming_Contracts`.
